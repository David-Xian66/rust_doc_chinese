展开描述

表示已生成到事件循环上的子进程。

§Caveats

与标准库的行为类似，且与 futures 范式中"丢弃即取消"不同， 默认情况下，已生成的进程即使在 Child 句柄被丢弃后仍将继续执行。

Command::kill_on_drop 方法可用于修改此行为， 在 Child 包装器在子进程退出前被丢弃时杀死子进程。

字段§

§stdin: Option<ChildStdin>

子进程标准输入 (stdin) 的写入 handle（若已捕获）。为避免部分移动 child 导致在使用 stdin 时无法对 child 调用其他函数，你可能需要这样做:

let stdin = child.stdin.take().unwrap();
§stdout: Option<ChildStdout>

子进程标准输出 (stdout) 的读取 handle（若已捕获）。 You might find it helpful to do

let stdout = child.stdout.take().unwrap();

这可以避免部分移动 child，从而在使用 stdout 时不会阻碍对 child 函数的调用。

§stderr: Option<ChildStderr>

子进程标准错误 (stderr) 的读取 handle（若已捕获）。 You might find it helpful to do

let stderr = child.stderr.take().unwrap();

这可以避免部分移动 child，从而在使用 stderr 时不会阻碍对 child 函数的调用。

pub fn id(&self) -> Option<u32>

返回与此子进程关联的、由操作系统分配的进程标识符（在子进程仍在运行时）。

一旦子进程被轮询到完成，此方法将返回 None。 这样做是为了避免在 Unix 等操作系统上引起混淆——这些平台上 进程完成后，操作系统标识符可能会被回收复用。

pub fn start_kill(&mut self) -> Result<()>

尝试强制子进程退出，但不等待请求生效。

On Unix platforms, this is the equivalent to sending a SIGKILL. Note 在 Unix 平台上发送 kill 信号后子进程可能成为僵尸进程；为避免这种情况，调用者应确保 either child.wait().await or child.try_wait() is invoked successfully.

Source

pub async fn kill(&mut self) -> Result<()>

强制子进程退出。

这相当于在 unix 平台上发送 SIGKILL， 然后调用 wait。

注意：标准库版本的 Child::kill 不会 wait。 在标准库中，如需等价于 Child::kill 的功能， 请使用 start_kill。

§示例

如果需要远程杀死子进程，可以使用 select! 宏 结合 oneshot 信道来实现。在以下示例中，子进程将运行到完成， 除非在 oneshot 信道上发送消息。 如果发送了消息，则会立即使用 .kill() 方法杀死子进程。

use tokio::process::Command;
use tokio::sync::oneshot::channel;

#[tokio::main]
async fn main() {
    let (send, recv) = channel::<()>();
    let mut child = Command::new("sleep").arg("1").spawn().unwrap();
    tokio::spawn(async move { send.send(()) });
    tokio::select! {
        _ = child.wait() => {}
        _ = recv => child.kill().await.expect("kill failed"),
    }
}

你还可以与子进程的标准 I/O 交互。例如， 可以在等待子进程退出的同时读取其 stdout。


#[tokio::main]
async fn main() {
    let (_tx, rx) = channel::<()>();

    let mut child = Command::new("echo")
        .arg("Hello World!")
        .stdout(Stdio::piped())
        .spawn()
        .unwrap();

    let mut stdout = child.stdout.take().expect("stdout is not captured");

    let read_stdout = tokio::spawn(async move {
        let mut buff = Vec::new();
        let _ = stdout.read_to_end(&mut buff).await;

        buff
    });

    tokio::select! {
        _ = child.wait() => {}
        _ = rx => { child.kill().await.expect("kill failed") },
    }

    let buff = read_stdout.await.unwrap();

    assert_eq!(buff, b"Hello World!\n");
}

pub async fn wait(&mut self) -> Result<ExitStatus>

等待子进程完全退出，返回它退出的状态。此函数至少被调用一次后，将继续返回相同的返回值。

子进程的 stdin 句柄（如果有）在等待之前会被关闭。 这有助于避免死锁：它确保子进程不会因等待父进程的输入而阻塞， 而父进程则在等待子进程退出。

若调用者希望显式控制子进程的 stdin handle 何时关闭，可在调用 .wait() 之前先 .take()：

§Cancel safety

此函数是取消安全的。

use tokio::io::AsyncWriteExt;
use tokio::process::Command;
use std::process::Stdio;

#[tokio::main]
async fn main() {
    let mut child = Command::new("cat")
        .stdin(Stdio::piped())
        .spawn()
        .unwrap();

    let mut stdin = child.stdin.take().unwrap();
    tokio::spawn(async move {
        // do something with stdin here...
        stdin.write_all(b"hello world\n").await.unwrap();

        // then drop when finished
        drop(stdin);
    });

    // wait for the process to complete
    let _ = child.wait().await;
}

pub fn try_wait(&mut self) -> Result<Option<ExitStatus>>

若子进程已退出，尝试收集其退出状态。

此函数不会阻塞调用线程，只会检查子进程是否已退出。 如果子进程已退出，则在 Unix 上会回收其进程 ID。 只要子进程已经退出，此函数保证可以重复返回成功的退出状态。

若子进程已退出，返回 Ok(Some(status))。若退出状态此时不可用，返回 Ok(None)。若发生错误，返回该错误。

请注意，与 wait 不同，此函数不会尝试关闭 stdin，也不会在子进程退出时唤醒当前任务。

pub async fn wait_with_output(self) -> Result<Output>

返回一个 Future，它将解析为 Output，包含子进程的退出状态、stdout 与 stderr。

返回的 Future 将同时等待子进程退出并收集 stdout/stderr handle 上所有剩余输出，返回一个 Output 实例。

子进程的 stdin handle（若有）在等待前将被关闭。这有助于避免死锁：它确保子进程不会因等待父进程输入而阻塞，同时父进程正在等待子进程退出。

默认情况下，stdin、stdout 与 stderr 继承自父进程。要将输出捕获到此 Output，需要在父进程与子进程之间创建新管道。 Use stdout(Stdio::piped()) or stderr(Stdio::piped()), respectively, when creating a Command.
