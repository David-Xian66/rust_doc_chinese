函数 block_in_place 

Source 
pub fn block_in_place<F, R>(f: F) -> R
where
    F: FnOnce() -> R,
展开描述

在当前线程上运行提供的阻塞函数， 而不阻塞执行器。

一般来说，发出阻塞调用或在 future 中执行大量计算而不让出是有问题的， 因为它可能会阻止执行器 推进其他任务。调用此函数 通知执行器当前正在执行的任务即将阻塞线程， 以便执行器能够在此之前 将其拥有的任何其他任务移交给 新的工作线程。有关更多信息， 请参阅CPU 密集型任务和阻塞代码部分。

请注意，尽管此函数避免了饿死其他独立 生成的任务，但在调用 block_in_place 期间， 同一任务中并发运行的任何其他代码 都将被挂起。例如在使用 join! 宏时 可能发生这种情况。为避免此问题， 请使用 spawn_blocking 代替 block_in_place。

请注意，此函数不能在 current_thread 运行 时内使用，因为在这种情况下 没有其他工作线程可以移交任务。 另一方面，允许在运行时外部调用此函数。 在这种情况下，block_in_place 只是正常地 调用所提供的闭包。

block_in_place 后运行的代码 无法被取消。当你关闭 执行器时，它将无限期等待所有阻塞操作完成。 你可以使用 shutdown_timeout 在 超时后停止等待它们。请注意，这 仍然不会取消这些任务â它们 只是在方法返回后继续运行。

§示例

use tokio::task;

task::block_in_place(move || {
    // do some compute-heavy work or call synchronous code
});

block_in_place 内运行的代码可以使用 block_on 重新进入 async 上下文。

use tokio::task;
use tokio::runtime::Handle;

task::block_in_place(move || {
    Handle::current().block_on(async move {
        // do something async
    });
});

§Panics

如果从 current_thread 运行时调用，此函数会 panic。
