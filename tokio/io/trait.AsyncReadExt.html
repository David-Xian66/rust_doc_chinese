展开描述

从源读取字节。

作为扩展 trait 实现，向所有 AsyncRead 类型添加实用方法。 调用者通常会导入此 trait 而不是 AsyncRead。

use tokio::fs::File;
use tokio::io::{self, AsyncReadExt};

#[tokio::main]
async fn main() -> io::Result<()> {
    let mut f = File::open("foo.txt").await?;
    let mut buffer = [0; 10];

    // The `read` method is defined by this trait.
    let n = f.read(&mut buffer[..]).await?;

    Ok(())
}

有关更多详细信息，请参阅 模块 文档。

fn chain<R>(self, next: R) -> Chain<Self, R>
where Self: Sized, R: AsyncRead,

创建一个新的 AsyncRead 实例，将此流与 next 链接起来。

返回的 AsyncRead 实例将首先从此对象读取所有字节直到 EOF。之后的输出等同于 next 的输出。

§示例

File 实现 AsyncRead：

use tokio::fs::File;
use tokio::io::{self, AsyncReadExt};

#[tokio::main]
async fn main() -> io::Result<()> {
    let f1 = File::open("foo.txt").await?;
    let f2 = File::open("bar.txt").await?;

    let mut handle = f1.chain(f2);
    let mut buffer = String::new();

    // read the value into a String. We could use any AsyncRead
    // method here, this is just one example.
    handle.read_to_string(&mut buffer).await?;
    Ok(())
}

fn read<'a>(&'a mut self, buf: &'a mut [u8]) -> Read<'a, Self>
where Self: Unpin,

从此源拉取一些字节到指定的缓冲中，返回读取的字节数。

等价于：

ⓘ
async fn read(&mut self, buf: &mut [u8]) -> io::Result<usize>;

此方法不保证是立即完成还是异步完成。

§Return

若此方法的返回值为 Ok(n)，则必须保证 0 <= n <= buf.len()。非零的 n 值表示缓冲 buf 已从该源填充了 n 个字节的数据。若 n 为 0，则可能表示以下两种情况之一：

  1. This reader has reached its “end of file” and will likely no longer be able to produce bytes. Note that this does not mean that the reader will always no longer be able to produce bytes.
  2. The buffer specified was 0 bytes in length.

调用此函数时，不保证 buf 的内容，实现不能依赖 buf 内容的任何属性。建议实现仅向 buf 写入数据，而不是读取其内容。

相应地，但是，此方法的调用者不能假设实现使用 buf 的方式的任何保证。打算写入缓冲的代码也可能从其中读取。 请你负责确保在调用 read 之前 buf 已初始化。

§错误

若此函数遇到任何形式的 I/O 或其他错误，则返回错误变体。若返回错误，则必须保证尚未读取任何字节。

§Cancel safety

此方法是取消安全的。若你将其作为事件用在 tokio::select! 语句中，且其他分支先完成，则保证尚未读取任何数据。

§示例

File 实现 Read：

use tokio::fs::File;
use tokio::io::{self, AsyncReadExt};

#[tokio::main]
async fn main() -> io::Result<()> {
    let mut f = File::open("foo.txt").await?;
    let mut buffer = [0; 10];

    // read up to 10 bytes
    let n = f.read(&mut buffer[..]).await?;

    println!("The bytes: {:?}", &buffer[..n]);
    Ok(())
}

fn read_buf<'a, B>(&'a mut self, buf: &'a mut B) -> ReadBuf<'a, Self, B>
where Self: Unpin, B: BufMut + ?Sized,

从此源拉取一些字节到指定的缓冲中，前进缓冲的内部游标。

等价于：

ⓘ
async fn read_buf<B: BufMut>(&mut self, buf: &mut B) -> io::Result<usize>;

通常，即使提供的缓冲还有更多空间，也只会发出单个 read 系统调用。

此方法不保证是立即完成还是异步完成。

§Return

非零的 n 值表示缓冲 buf 已从该源填充了 n 个字节的数据。若 n 为 0，则可能表示以下两种情况之一：

  1. This reader has reached its “end of file” and will likely no longer be able to produce bytes. Note that this does not mean that the reader will always no longer be able to produce bytes.
  2. The buffer specified had a remaining capacity of zero.
§错误

若此函数遇到任何形式的 I/O 或其他错误，则返回错误变体。若返回错误，则必须保证尚未读取任何字节。

§Cancel safety

此方法是取消安全的。若你将其作为事件用在 tokio::select! 语句中，且其他分支先完成，则保证尚未读取任何数据。

§示例

File 实现了 Read，BytesMut 实现了 BufMut：

use tokio::fs::File;
use tokio::io::{self, AsyncReadExt};

use bytes::BytesMut;

#[tokio::main]
async fn main() -> io::Result<()> {
    let mut f = File::open("foo.txt").await?;
    let mut buffer = BytesMut::with_capacity(10);

    assert!(buffer.is_empty());
    assert!(buffer.capacity() >= 10);

    // note that the return value is not needed to access the data
    // that was read as `buffer`'s internal cursor is updated.
    //
    // this might read more than 10 bytes if the capacity of `buffer`
    // is larger than 10.
    f.read_buf(&mut buffer).await?;

    println!("The bytes: {:?}", &buffer[..]);
    Ok(())
}

fn read_exact<'a>(&'a mut self, buf: &'a mut [u8]) -> ReadExact<'a, Self>
where Self: Unpin,

读取填充 buf 所需的精确字节数。

等价于：

ⓘ
async fn read_exact(&mut self, buf: &mut [u8]) -> io::Result<usize>;

此函数读取必要的字节数，以完全填充指定的缓冲 buf。

§错误

若操作在完全填满缓冲之前遇到“文件结束”，则返回 ErrorKind::UnexpectedEof 类型的错误。这种情况下 buf 的内容是未指定的。

若遇到任何其他读取错误，操作将立即返回。这种情况下 buf 的内容是未指定的。

若此操作返回错误，则读取的字节数是未指定的，但绝不会读取超过完全填充缓冲所需的字节数。

§Cancel safety

此方法不是可取消安全的。如果在 tokio::select! 语句中作为事件使用且另一个分支先完成， 则某些数据可能已经 被读入 buf。

§示例

File 实现 Read：

use tokio::fs::File;
use tokio::io::{self, AsyncReadExt};

#[tokio::main]
async fn main() -> io::Result<()> {
    let mut f = File::open("foo.txt").await?;
    let len = 10;
    let mut buffer = vec![0; len];

    // read exactly 10 bytes
    f.read_exact(&mut buffer).await?;
    Ok(())
}

fn read_to_end<'a>(&'a mut self, buf: &'a mut Vec<u8>) -> ReadToEnd<'a, Self>
where Self: Unpin,

从此源读取所有字节直到 EOF，将它们放入 buf。

等价于：

ⓘ
async fn read_to_end(&mut self, buf: &mut Vec<u8>) -> io::Result<usize>;

从此源读取的所有字节都将附加到指定的 缓冲区 buf。此函数会持续调用 read() 来 将更多数据附加到 buf，直到 read() 返回 Ok(0)。

如果成功，则返回读取的总字节数。

§错误

若遇到读取错误，read_to_end 操作将立即完成。已读取的所有字节将追加到 buf。

§示例

File 实现 Read：

use tokio::io::{self, AsyncReadExt};
use tokio::fs::File;

#[tokio::main]
async fn main() -> io::Result<()> {
    let mut f = File::open("foo.txt").await?;
    let mut buffer = Vec::new();

    // read the whole file
    f.read_to_end(&mut buffer).await?;
    Ok(())
}

（另请参阅用于从文件读取的便捷函数 tokio::fs::read。）

fn read_to_string<'a>( &'a mut self, dst: &'a mut String, ) -> ReadToString<'a, Self>
where Self: Unpin,

从此源读取所有字节直到 EOF，将它们追加到 buf。

等价于：

fn take(self, limit: u64) -> Take<Self>
where Self: Sized,

创建一个适配器，从其中最多读取 limit 个字节。

此函数返回 AsyncRead 的一个新实例，最多读取 limit 个字节，之后将始终返回 EOF（Ok(0)）。 Any read errors will not count towards the number of bytes read and future calls to read() may succeed.

§示例

File 实现 Read：

use tokio::io::{self, AsyncReadExt};
use tokio::fs::File;

#[tokio::main]
async fn main() -> io::Result<()> {
    let f = File::open("foo.txt").await?;
    let mut buffer = [0; 5];

    // read at most five bytes
    let mut handle = f.take(5);

    handle.read(&mut buffer).await?;
    Ok(())
}

动态兼容性§

此 trait 不是dyn 兼容的。

在较早版本的 Rust 中，dyn 兼容性被称为"对象安全"，因此此 trait 不是对象安全的。
