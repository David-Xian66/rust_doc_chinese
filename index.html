Rust 文档中文翻译 

展开描述

本项目是若干 Rust crate 的官方 rustdoc 文档的中文翻译版。覆盖异步运行时、网络协议、TLS、多媒体捕获、键鼠模拟等常用领域。

支持两种搜索：

  • 顶栏「搜索」按钮：按类型、方法、字段等标识符搜索（原生 rustdoc 搜索）
  • 顶栏「搜索文档」按钮：按已翻译的中文注释、模块说明搜索（自建 MiniSearch 索引）

按 S 或 / 可快速触发搜索。

数据结构§

bytes
bytes 是 Rust 生态常用的零拷贝字节缓冲区与字符串切片容器，被 tokio / quinn / hyper 等网络栈广泛使用，是 I/O 处理的底层基础设施。

异步运行时§

tokio
Tokio 是 Rust 的异步运行时
tokio_macros
tokio 的过程宏集合，提供 #[main] / #[test] 入口宏与 select! / join! / try_join! / pin! / task_local! 五个核心组合宏，让异步代码的书写更简洁。
，提供事件循环、调度器、定时器等基础设施。本翻译基于 tokio 官方文档，覆盖 runtime / sync / io / net / task / time 等所有子模块。

网络与 TLS§

quinn
QUIC 协议实现，提供高性能、低延迟的 UDP 传输层。
rustls_pki_types
rustls 生态中 X.509 证书、密钥等类型的统一定义。被 rustls、rustls-webpki 等 crate 使用。
rustls
纯 Rust 编写的现代 TLS 库，提供客户端与服务器实现，支持 TLS 1.2 / 1.3 与 QUIC。
rcgen
X.509 证书生成库，常用于测试、TLS 服务端本地签发证书、构建 PKI 测试场景等。

多媒体与捕获§

windows_capture
Windows 屏幕捕获 API 绑定（基于 Direct3D 11 与 DXGI Duplication API）。

键鼠模拟/监听§

enigo
跨平台键鼠模拟库，支持 Windows / macOS / Linux。
rdev
跨平台键鼠事件监听与模拟库。

FFI 绑定§

ffmpeg_next
FFmpeg 多媒体框架的 Rust 绑定（高层 API）。
ffmpeg_sys_next
FFmpeg 的底层 FFI 绑定（unsafe API）。

所有 crate§

windows
Windows API 的 Rust 绑定（顶层 umbrella crate）。
windows_core
Windows API 的核心绑定类型与 trait。
scrap
屏幕捕获库（跨平台，当前文档为英文原文）。
